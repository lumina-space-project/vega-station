## 💻 2. Стандарты разработки

### 🦀 Код (C# / Robust Toolbox)
*   **Стиль:** Мы следуем [стандартам кодирования SS14](https://docs.spacestation14.com/en/general-development/codebase-info/codebase-organization.html).
*   **Чистота:** Код должен быть адекватно задокументирован. Рекомендуется использовать YAML Linter.
*   **Форматирование:** Используйте файлы `.editorconfig`, включенные в репозиторий.
*   **Атомарность:** Один коммит = одна задача. Не смешивайте исправление бага и добавление новой фичи в одной записи.
*   **Проверка изменений:** Всегда проверяйте результат в игре (локальный сервер)

## 🚀 3. Процесс внесения изменений

1. **Создание Issue:** 📝 Перед началом работы над крупной фичей, пожалуйста, создайте обсуждение (Issue) на github, либо в соответствующем канале [Discord-сервера](https://discord.gg/QU6s5xnyRe), чтобы избежать дублирования работы.
2. **Ветвление:** 🌿 Создавайте отдельную ветку для каждой задачи используя структуру со слэшем `/` для их группировки.
    - **Формат:** `тип/краткое-описание` (на английском, строчными буквами).
    - *Примеры:*
        - `feat/new-sindi-implants` — добавление нового функционала
        - `fix/l10n-typo-header` — исправление текста.
        - `refactor/clean-code` — переработка кода без смены логики.
3. **Коммиты:** 📑 Мы придерживаемся стандарта **[Conventional Commits](https://conventionalcommits.org)**.

    - Основные типы (Type) коммитов:
        - feat — новая функциональность (добавили язык, новую кнопку, эндпоинт).
        - fix — исправление ошибки (поправили опечатку, баг в логике).
        - refactor — правка кода без изменения логики (улучшение читаемости кода).
        - docs — всё, что связано с документацией.
        - style — правки оформления (пробелы, отступы), не влияющие на код.

    - Область (Scope) - помогает понять, какую часть системы затрагивает изменение
        - **Формат:** `тип(область): описание с маленькой буквы` (без точки в конце).
        - **Примеры:**
        - `feat(ui): добавлена кнопка переключения языка в лобби`
        - `fix(core): исправлен вылет сервера при загрузке пустых файлов перевода`
        - `feat(auth): добавлена авторизация через Google`

4. **Pull Request (PR):** 📥
    * Проверьте ваш код на следование стандартам разработки, указанным выше.
    * Убедитесь, что проект собирается без ошибок.
    * При отправке Pull Request корректно заполните соответствующий шаблон.
